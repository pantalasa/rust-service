# Contributing

Thanks for contributing to this service.

## Process

1. Create a feature branch from the default branch.
2. Make your changes and add tests where appropriate.
3. Run the test suite locally and ensure it passes.
4. Open a pull request describing the change and its motivation.

## Conventions

- Follow the language's standard style and keep modules small and focused.
- Keep pull requests scoped and reviewable.
- Reference a ticket in the PR title where applicable.

## Code of Conduct

Be respectful and constructive. Report concerns to engineering@pantalasa.org.
